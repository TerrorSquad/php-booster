We include a pre-configured [Renovate](https://docs.renovatebot.com/) setup for automated dependency management.

## Configuration

The default [renovate.json](https://github.com/TerrorSquad/php-booster/blob/main/booster/renovate.json) configuration includes:

```json
{
  "$schema": "https://docs.renovatebot.com/renovate-schema.json",
  "extends": [
    "config:base"
  ],
  "packageRules": [
    {
      "matchUpdateTypes": ["minor", "patch"],
      "matchCurrentVersion": "!/^0/",
      "automerge": true
    },
    {
      "matchDepTypes": ["devDependencies"],
      "groupName": "dev dependencies",
      "schedule": ["every weekend"]
    },
    {
      "matchPackagePatterns": ["phpunit/", "phpstan/", "psalm/", "symfony/", "doctrine/", "guzzlehttp/"],
      "groupName": "PHP dependencies"
    }
  ],
  "timezone": "Europe/Paris",
  "schedule": ["every weekend"],
  "labels": ["dependencies", "renovate"],
  "branchPrefix": "deps/"
}
```

### Key Settings

::field-group
  ::field{name="extends" type="string" defaultValue="config:base"}
  Base configuration preset
  ::
  ::field{name="schedule" type="string" defaultValue="every weekend"}
  When to check for updates
  ::
  ::field{name="timezone" type="string" defaultValue="Europe/Paris"}
  Timezone for scheduling
  ::
  ::field{name="labels" type="array" defaultValue="dependencies, renovate"}
  PR labels
  ::
  ::field{name="branchPrefix" type="string" defaultValue="deps/"}
  Branch naming prefix
  ::
  ::field{name="automerge" type="boolean" defaultValue="true (patch/minor)"}
  Auto-merge stable updates
  ::
::

### Customization

::note
You can customize the configuration after integration by editing `renovate.json` in your project root:
::

#### Change Update Schedule
```json
{
  "schedule": ["every weekday"]
}
```

#### Adjust Automerge Rules
```json
{
  "packageRules": [
    {
      "matchUpdateTypes": ["patch"],
      "automerge": true
    }
  ]
}
```

#### Customize Grouping
```json
{
  "packageRules": [
    {
      "matchPackagePatterns": ["symfony"],
      "groupName": "Symfony packages"
    }
  ]
}
```

## Enabling Renovate

### On GitHub

::steps
  ### Install the App
  Install the [Renovate GitHub App](https://github.com/apps/renovate) on your repository

  ### Grant Permissions
  Grant necessary permissions (read repo contents, create PRs)

  ### Automatic Detection
  Renovate will automatically detect your `renovate.json` configuration

  ### Wait for Updates
  Wait for the first pull request with dependency updates
::

### On GitLab

::steps
  ### Go to Settings
  Go to your project's **Settings > Members**

  ### Add Renovate Bot
  Add the Renovate bot user with at least **Developer** role (required for creating merge requests)

  ### Configure Variables
  Configure CI/CD variables if needed:
   - `RENOVATE_TOKEN`: Personal access token with API access (for private repos)
   - `RENOVATE_PLATFORM`: Set to `gitlab`

  ### Schedule Pipelines (Optional)
  Optional: Set up scheduled pipelines to run Renovate at regular intervals

  ### Start
  Renovate will start creating merge requests based on your `renovate.json` configuration
::

For more details, see [Renovate GitLab documentation](https://docs.renovatebot.com/modules/platform/gitlab/).

### Self-Hosted

For self-hosted Renovate installations:

::steps
  ### Install Renovate
  Install Renovate following the [self-hosting guide](https://docs.renovatebot.com/self-hosting/)

  ### Configure Instance
  Configure your Renovate instance to scan repositories with `renovate.json` files

  ### Set Environment Variables
  Set up environment variables:
   - `RENOVATE_PLATFORM`: Your git platform (`github`, `gitlab`, `bitbucket`, etc.)
   - `RENOVATE_ENDPOINT`: Your platform's API endpoint
   - `RENOVATE_TOKEN`: Authentication token with repository access

  ### Verify Configuration
  The provided `renovate.json` configuration works with self-hosted instances

  ### Customize Schedule
  Consider customizing the schedule to match your team's workflow
::

**Example Docker command:**
```bash
docker run --rm \
  -e RENOVATE_PLATFORM=gitlab \
  -e RENOVATE_ENDPOINT=https://gitlab.example.com/api/v4 \
  -e RENOVATE_TOKEN=your-token \
  -v /path/to/config:/usr/src/app/config \
  renovate/renovate
```

For more advanced setups, refer to the [official self-hosting documentation](https://docs.renovatebot.com/self-hosting/).

## Troubleshooting

### Renovate Not Creating PRs
- Verify the Renovate app is installed and has access
- Check that `renovate.json` is valid JSON
- Review Renovate logs in your repository settings
- Ensure dependencies are not pinned or locked

### Too Many PRs
- Adjust the schedule to less frequent checks
- Enable more aggressive grouping rules
- Increase the automerge criteria

### Failing Status Checks
- Review your CI/CD pipeline configuration
- Check if tests need updates for new package versions
- Consider updating test fixtures or mocks

### Merge Conflicts
- Keep your base branch up to date
- Rebase Renovate branches regularly
- Enable Renovate's rebase options
