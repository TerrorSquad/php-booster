We use [PHP Security Checker](https://github.com/enlightn/security-checker) to detect known security vulnerabilities in your dependencies.

## PHP Security Checker

This tool checks your `composer.lock` file against the [Security Advisories Database](https://github.com/fabpot/local-php-security-checker).

### Usage

The security check runs automatically on `pre-commit` when you modify `composer.lock`.

You can also run it manually:

::code-group
  ```bash [Standard]
  vendor/bin/security-checker security:check
  ```

  ```bash [DDEV]
  ddev vendor/bin/security-checker security:check
  ```
::

### Configuration

This tool does not typically require configuration. It checks the currently installed versions in `composer.lock`.

### Ignoring Vulnerabilities

If you need to ignore a specific vulnerability, you can use the `--ignore` flag, but it is generally recommended to update the vulnerable package instead.
