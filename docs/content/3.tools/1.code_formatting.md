We use [EasyCodingStandard (ECS)](https://github.com/easy-coding-standard/easy-coding-standard) for PHP and [Prettier](https://prettier.io/) for other files to enforce consistent code style and formatting.

## EasyCodingStandard (ECS)

### Usage

Use the following commands, which are also defined as Composer scripts:

::code-group
  ```bash [Standard]
  # Check style
  composer check-cs
  
  # Fix style
  composer fix-cs
  ```

  ```bash [DDEV]
  # Check style
  ddev composer check-cs
  
  # Fix style
  ddev composer fix-cs
  ```
::

## Configuration

The [ecs.php](https://github.com/TerrorSquad/php-booster/blob/main/booster/ecs.php) file in your project root is where you configure EasyCodingStandard.

## Markdownlint

We use [markdownlint-cli2](https://github.com/DavidAnson/markdownlint-cli2) to lint Markdown files.

It checks for common issues and helps maintain consistent documentation style.

### Usage

This tool runs automatically on `pre-commit` when you modify Markdown files.

### Configuration

You can configure rules by adding a `.markdownlint.json` or `.markdownlint.jsonc` file to your project root.
