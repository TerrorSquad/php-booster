We use [PHPStan](https://phpstan.org/) and [Psalm](https://psalm.dev/) for static analysis to catch errors and enforce type safety.

## PHPStan

### Usage

::code-group
  ```bash [Standard]
  composer phpstan 
  ```

  ```bash [SonarQube]
  composer phpstan:sonar
  ```

  ```bash [DDEV]
  ddev composer phpstan
  ddev composer phpstan:sonar
  ```
::

### Configuration
The [phpstan.neon.dist](https://github.com/TerrorSquad/php-booster/blob/main/booster/phpstan.neon.dist) file in your project root provides the base configuration. Create a `phpstan.neon` file to override settings.

## Psalm

### Usage

::code-group
  ```bash [Standard]
  composer psalm
  ```

  ```bash [SonarQube]
  composer psalm:sonar
  ```

  ```bash [DDEV]
  ddev composer psalm
  ddev composer psalm:sonar
  ```
::

* **In your IDE:** See the [IDE Configuration](/tools/ide_configuration#ddev-integration-helper) page for details on setting up the `psalm-ddev` wrapper.

### Configuration

The [psalm.xml](https://github.com/TerrorSquad/php-booster/blob/main/booster/psalm.xml) file in your project root configures Psalm.
